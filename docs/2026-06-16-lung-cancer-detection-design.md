# Lung Cancer Detection from Chest CT — Design Spec

**Date:** 2026-06-16
**Authors of the project:** Azizbek Oqbutayev, Nozima Sotiboldiyeva
**Built with:** Claude Code (Opus 4.8)

## 1. Goal

Train an AI model to detect lung cancer from chest CT scan images, document every
step in a Jupyter ("Anaconda") notebook, and ship a static website that (a) presents
the results visually, (b) credits the authors, and (c) lets a visitor test the model
live in the browser.

## 2. Dataset decision

The literal first dataset in `dataset.md` is **LIDC-IDRI** (~125 GB of 3-D CT volumes,
TCIA downloader, DICOM, 3-D CNNs on a GPU). That is infeasible to download/train in
this environment and cannot be embedded in a lightweight web demo.

**Chosen dataset:** the **Chest CT-Scan images** set (cited in `dataset.md` §B,
originally Kaggle/`mohamedhanyyy`), mirrored on Hugging Face as `dorsar/lung-cancer`.

- Real chest CT slices, JPG/PNG, pre-split into `train` / `valid` / `test`.
- 4 classes: **adenocarcinoma, large cell carcinoma, squamous cell carcinoma, normal**.
- ~767 images total (train balanced at 100/class; realistic imbalanced test set).
- License: **MIT** — which doubles as a live teaching example of the exact "MIT"
  confusion that `dataset.md` debunks (MIT *License* ≠ MIT *the institution*).
- Small enough to train fast on the M4 via transfer learning, small enough to export
  to a browser-runnable model.

Fallback (only if needed): tabular survey CSV with in-browser JS inference.

## 3. Model

- **Transfer learning** with a MobileNetV2 backbone pretrained on ImageNet.
- Backbone frozen; a new classifier head trained for the 4 CT classes.
- Rationale: works well on small data, exports cleanly to ONNX, and is small/fast
  enough to run client-side in the browser.
- Loss: cross-entropy (class-weighted for the imbalanced test reality). Optimizer: Adam.
- Device: Apple MPS if available, else CPU.

## 4. Components

```
lung-cancer-detection/
  docs/        this spec
  data/        downloaded Chest CT dataset (chest-ct/Data/{train,valid,test}/<class>)
  notebooks/   lung_cancer_detection.ipynb  <- the executed documentation artifact
  models/      best_model.pt, model.onnx
  website/     index.html, style.css, app.js, model.onnx, labels/preproc/metrics JSON
    assets/    figures generated by the notebook (EDA, curves, confusion matrix, etc.)
```

Data flow: notebook trains -> saves figures + `metrics.json` to `website/assets/`,
exports `model.onnx` + `labels.json` + `preprocessing.json` to `website/` -> the static
site reads those JSON/PNG artifacts and runs the ONNX model in-browser via
`onnxruntime-web`.

## 5. Notebook (documents everything)

1. Title + authors + overview
2. Background — the "no MIT lung-cancer dataset" finding; MIT License vs MIT institution
3. Environment & reproducibility (seeds, device)
4. Data loading + class-label normalization (train folders carry TNM-staging suffixes)
5. EDA — class distribution, sample images per class
6. Preprocessing & augmentation rationale
7. Model architecture (transfer learning)
8. Training loop with per-epoch validation; save best
9. Training curves
10. Evaluation — accuracy, per-class precision/recall/F1, confusion matrix, one-vs-rest
    ROC/AUC, plus a clinically-framed binary "cancer vs normal" sensitivity/specificity
11. Sample predictions gallery
12. Grad-CAM — where the model looks
13. ONNX export + PyTorch↔ONNX parity check
14. Results summary, limitations, authors, references

## 6. Website

Static (no backend). Sections: hero (title + authors), the MIT-myth callout,
dataset card, methodology, interactive results (metrics, confusion matrix, ROC,
Grad-CAM, sample gallery), and a **"Test the AI"** drag-and-drop box running the ONNX
model in-browser. Prominent medical disclaimer (educational use only, not diagnostic).

## 7. Honesty / limitations (carried into the deliverables)

- Small, single-source dataset; CT slices not full volumes.
- Educational demonstration, **not** a clinical tool.
- Browser preprocessing approximates the Python pipeline (resize interpolation differs
  slightly); a parity check is reported in the notebook.

## 8. Cost

$0 — local conda env, free HF dataset, free pretrained weights, static site,
in-browser inference (no server, no per-token API).
